# خطة بناء نظام ذكي لاكتشاف الحرائق وإطفائها (IoT Fire Detection & Suppression System)

نظام يعتمد على **ESP32** لاكتشاف الحريق عبر حساس اللهب، ويستجيب تلقائياً بتشغيل إنذار (LED + Buzzer) ومضخة مياه عبر ريليه، مع مراقبة وتحكم عن بُعد من تطبيق الهاتف عبر Wi-Fi.

---

## 1. المكونات المطلوبة (Hardware)

| المكوّن | الوصف | العدد |
|---|---|---|
| ESP32 DevKit V1 | المتحكم الرئيسي مع Wi-Fi مدمج | 1 |
| حساس اللهب (Flame Sensor IR) | يكشف الأشعة تحت الحمراء الصادرة من اللهب (خرج رقمي + تناظري) | 1 (أو 3 لتغطية أوسع) |
| وحدة ريليه 5V (قناة واحدة) | لتشغيل/إيقاف المضخة | 1 |
| مضخة مياه صغيرة (5V–12V DC) | للإطفاء | 1 |
| Buzzer نشط | إنذار صوتي | 1 |
| LED أحمر + LED أخضر | مؤشر خطر / مؤشر حالة طبيعية | 2 |
| مقاومات 220Ω | لحماية الـ LED | 2 |
| مصدر تغذية خارجي | للمضخة (لا تُغذّى من ESP32 مباشرة!) | 1 |
| Breadboard + أسلاك توصيل | للتجميع الأولي | — |
| خرطوم ماء صغير + خزان | لنظام الرش | — |

> **اختياري للتطوير:** حساس دخان MQ-2 وحساس حرارة DHT22 لتقليل الإنذارات الكاذبة (تأكيد الحريق من أكثر من مصدر).

---

## 2. مخطط التوصيل (Wiring)

| الطرف | ESP32 Pin |
|---|---|
| Flame Sensor DO | GPIO 34 (رقمي) |
| Flame Sensor AO | GPIO 35 (تناظري - اختياري لقياس شدة اللهب) |
| Relay IN | GPIO 26 |
| Buzzer | GPIO 27 |
| LED أحمر (إنذار) | GPIO 25 |
| LED أخضر (حالة طبيعية) | GPIO 33 |
| VCC / GND | 3.3V أو 5V حسب الوحدة / GND مشترك |

**ملاحظات مهمة:**
- المضخة تُوصَّل عبر تلامسات الريليه (COM/NO) بمصدر تغذية خارجي، مع **GND مشترك** بين ESP32 ودائرة الريليه.
- أغلب وحدات الريليه تعمل بمنطق معكوس (Active LOW) — يجب مراعاة ذلك في الكود.
- GPIO 34 و35 للإدخال فقط (مناسبة للحساسات).

## 3. معمارية النظام (Architecture)

```
[حساس اللهب] ──► [ESP32] ──► [Relay ──► مضخة المياه]
                    │  └────► [LED + Buzzer]
                    │
                 Wi-Fi
                    │
              [MQTT Broker / Blynk Cloud]
                    │
              [تطبيق الهاتف]
        (مراقبة الحالة + تحكم يدوي بالمضخة)
```

**خيارات منصة الاتصال (اختر واحدة):**

| الخيار | المميزات | الأنسب لـ |
|---|---|---|
| **Blynk** (موصى به للنموذج الأولي) | تطبيق جاهز، لوحة تحكم سهلة، إشعارات Push بدون برمجة تطبيق | إنجاز سريع |
| MQTT (HiveMQ/Mosquitto) + تطبيق Flutter | مرونة كاملة، معيار صناعي | تعلم أعمق وتطبيق مخصص |
| Firebase + تطبيق مخصص | قاعدة بيانات فورية وسجل تاريخي | تخزين البيانات وتحليلها |

## 4. مراحل التنفيذ

### المرحلة 1: تجهيز بيئة العمل (يوم 1)
1. تثبيت Arduino IDE (أو PlatformIO مع VS Code — موصى به).
2. إضافة دعم لوحة ESP32 (Boards Manager).
3. اختبار اللوحة ببرنامج Blink بسيط.

### المرحلة 2: توصيل واختبار كل مكوّن على حدة (يوم 2–3)
1. قراءة حساس اللهب وطباعة القيم على Serial Monitor.
2. معايرة الحساس (المسافة، الحساسية عبر البوتنشيوميتر على الوحدة).
3. اختبار LED وBuzzer.
4. اختبار الريليه والمضخة (بمصدر تغذية خارجي).

### المرحلة 3: المنطق المحلي — الاستجابة التلقائية (يوم 4–5)
1. كتابة الكود الأساسي:
   - قراءة الحساس بشكل دوري.
   - عند اكتشاف لهب: تشغيل LED الأحمر + Buzzer + الريليه (المضخة).
   - عند زوال اللهب: إيقاف المضخة بعد مهلة أمان (مثلاً 5 ثوانٍ)، وإعادة LED الأخضر.
2. إضافة **Debounce/تأكيد** (اللهب يجب أن يستمر مثلاً 500ms قبل إطلاق الإنذار) لتقليل الإنذارات الكاذبة.
3. آلة حالات بسيطة: `NORMAL → FIRE_DETECTED → EXTINGUISHING → COOLDOWN → NORMAL`.

### المرحلة 4: الاتصال بالشبكة والتطبيق (يوم 6–8)
1. الاتصال بشبكة Wi-Fi مع إعادة اتصال تلقائي.
2. ربط النظام بـ Blynk (أو MQTT):
   - إرسال حالة النظام (طبيعي / حريق / إطفاء جارٍ).
   - إرسال إشعار فوري (Push Notification) عند اكتشاف حريق.
   - زر تحكم يدوي بالمضخة من التطبيق (تشغيل/إيقاف).
   - مفتاح وضع: تلقائي / يدوي.
3. **قاعدة أمان:** المنطق التلقائي يعمل محلياً على ESP32 حتى لو انقطع الإنترنت — الشبكة للمراقبة والتحكم الإضافي فقط.

### المرحلة 5: الاختبار والمعايرة (يوم 9–10)
1. اختبار الاكتشاف بمصدر لهب صغير آمن (ولاعة/شمعة) على مسافات مختلفة.
2. اختبار سيناريوهات: انقطاع Wi-Fi، انقطاع التيار، إنذار كاذب (ضوء شمس مباشر — حساسات IR تتأثر به).
3. قياس زمن الاستجابة من الاكتشاف حتى تشغيل المضخة.
4. اختبار التحكم اليدوي من التطبيق.

### المرحلة 6: التجميع النهائي والتوثيق (يوم 11–12)
1. نقل الدائرة من Breadboard إلى لوحة لحام أو PCB.
2. تركيب كل شيء في نموذج مصغّر (صندوق يمثّل غرفة).
3. توثيق المشروع: مخطط الدائرة، صور، فيديو تجربة، README.

## 6. الخوارزمية الأساسية (Pseudocode)

```
loop:
  flame = readFlameSensor()

  if mode == AUTO:
      if flame detected for >= 500ms:
          state = FIRE
          redLED ON, buzzer ON, pump ON
          sendNotification("🔥 حريق مكتشف!")
      else if state == FIRE and no flame for >= 5s:
          pump OFF, buzzer OFF
          state = NORMAL, greenLED ON

  if mode == MANUAL:
      pump = appPumpCommand   # من التطبيق

  publishStatus(state, flame, pump)   # كل 1-2 ثانية
```

## 7. اعتبارات السلامة ⚠️

- هذا **نموذج تعليمي** وليس بديلاً عن أنظمة إطفاء معتمدة.
- اختبر اللهب دائماً في بيئة آمنة ومفتوحة وبوجود طفاية يدوية.
- افصل دائرة المضخة (جهد أعلى) كهربائياً عن دائرة التحكم.
- **لا ترفع كلمات مرور Wi-Fi أو مفاتيح API إلى GitHub** — استخدم `secrets.h` مضافاً إلى `.gitignore`.

## 9. أفكار للتطوير المستقبلي

- إضافة حساس دخان (MQ-2) وحرارة (DHT22) واتخاذ القرار بدمج القراءات (Sensor Fusion).
- كاميرا ESP32-CAM لإرسال صورة عند الإنذار.
- بطارية احتياطية (UPS صغير) لاستمرار العمل عند انقطاع الكهرباء.
- عدة حساسات لهب لتغطية 360 درجة وتحديد اتجاه الحريق.
- سجل تاريخي للأحداث في قاعدة بيانات (Firebase/Supabase).
