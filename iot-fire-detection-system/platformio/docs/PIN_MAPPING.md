# Pin Mapping — جدول المنافذ

كل الأرقام مُعرّفة في مكان واحد: [`lib/constants/constants.h`](../lib/constants/constants.h). للانتقال إلى عتاد حقيقي، غالباً تغيّر هذه الأرقام فقط.

| المكوّن | الوظيفة | ESP32 GPIO | الاتجاه | ملاحظات |
|---|---|---|---|---|
| حساس اللهب (Flame DO) | كشف الحريق | **GPIO 34** | إدخال | Input-only pin؛ في المحاكاة يُمثّل بزر ضغط (`INPUT_PULLUP`) |
| وحدة الريليه (IN) | تشغيل المضخة | **GPIO 26** | إخراج | Active-LOW افتراضياً (انظر `RELAY_ACTIVE_LOW`) |
| Buzzer | إنذار صوتي | **GPIO 27** | إخراج | Buzzer نشط |
| LED أحمر | مؤشر الإنذار | **GPIO 25** | إخراج | عبر مقاومة 220Ω |
| LED أخضر | مؤشر الحالة الطبيعية | **GPIO 33** | إخراج | عبر مقاومة 220Ω |
| Wi-Fi | الاتصال | — (مدمج) | — | STA mode |

## Blynk Virtual Pins

| Virtual Pin | المحتوى | عنصر Blynk |
|---|---|---|
| V0 | حالة الحريق (0/1) | LED |
| V1 | حالة المضخة (0/1) | LED |
| V2 | الجهاز متصل (1) | LED |
| V3 | زر التحكم اليدوي بالمضخة | Button (Switch) |

## تنبيهات عتاد

- **المضخة تُغذّى من مصدر خارجي** عبر تلامسات الريليه (COM/NO)، لا من ESP32 مباشرة، مع GND مشترك.
- GPIO 34/35/36/39 للإدخال فقط (لا Pull-up داخلي على العتاد الحقيقي) — لكن حساس اللهب يقود الخط بنفسه فلا مشكلة.
- إن عملت المضخة بشكل معكوس، غيّر `RELAY_ACTIVE_LOW` في `constants.h`.
